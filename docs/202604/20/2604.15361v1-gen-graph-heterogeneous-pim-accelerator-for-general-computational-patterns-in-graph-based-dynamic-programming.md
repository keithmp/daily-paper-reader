## 摘要
虽然基于图的动态规划（DP）是基因组学和网络分析的基石，但其效率受到根本性冲突的计算模式的阻碍。以矩阵为中心的 DP 驱动规则的、计算受限的网络分析，而以拓扑为中心的 DP 则处理不规则的、内存受限的基因组遍历。这两类 DP 具有显著不同的计算模式和数据流，使得单一的同构存内计算（PIM）架构难以高效地同时支持两者。本研究提出了 GEN-Graph，一种新型异构 PIM 小芯片，它在 2.5D 封装内集成了两种类型的专用计算瓦片：Matrix-tile，一种针对以矩阵为中心的工作负载（如全源最短路径 APSP）优化的存内处理（PUM）瓦片；以及 traversal-tile，一种针对以遍历为中心的 DP 工作负载（如 DNA 序列比对）优化的近存处理（PNM）瓦片。我们的软硬件协同设计采用了递归划分和可重构窗口位并行逻辑，以确保精确计算。结果显示，在 APSP 任务中，matrix tile 相比 NVIDIA H100 GPU 实现了 42.8 倍的加速和 392 倍的能效提升。对于序列到图的比对，traversal tile 维持了每秒 256 万次读取（短读长）和每秒 3.93 万次读取（长读长），在吞吐量上优于现有最先进的加速器高达 2.56 倍。通过将硬件专业化与算法结构相匹配，GEN-Graph 为通用 DP 数据流提供了首个可扩展的精确解决方案。

## Abstract
While graph-based dynamic programming (DP) is a cornerstone of genomics and network analytics, its efficiency is hampered by fundamentally conflicting computational patterns. Matrix-centric DP drives regular, compute-bound network analytics, while topology-centric DP handles irregular, memory-bound genomic traversals. These two categories of DP have substantially different computation patterns and dataflows, which makes it difficult for a single homogeneous processing-in-memory (PIM) architecture to efficiently support both.   This work presents GEN-Graph, a novel heterogeneous PIM chiplet that integrates two types of specialized compute tiles within a 2.5D package: Matrix-tile, a processing-using-memory (PUM) tile optimized for matrix-centric workloads, such as all-pairs shortest path (APSP); and traversal-tile, a processing-near-memory (PNM) tile optimized for traversal-centric DP workloads, such as DNA sequence alignment. Our hardware-software co-design employs recursive partitioning and reconfigurable windowed bit-parallel logic to ensure exact computation. Results show the matrix tile achieves 42.8x speedup and 392x energy efficiency over the NVIDIA H100 GPU for APSP. For sequence-to-graph alignment, the traversal tile sustains 2.56 million reads/s (short-reads) and 39.3 thousand reads/s (long-reads), outperforming state-of-the-art accelerators by up to 2.56x in throughput. GEN-Graph provides the first scalable, exact solution for general DP dataflows by matching hardware specialization to algorithmic structure.

*   **核心问题**：基于图的动态规划（DP）在基因组学和网络分析中至关重要，但存在两种截然不同的计算模式冲突：
    *   **矩阵中心型（Matrix-centric）**：如全对最短路径（APSP），表现为规则的、计算密集型任务，瓶颈在于算术吞吐量。
    *   **拓扑中心型（Topology-centric）**：如序列到图（S2G）比对，表现为不规则的、访存密集型任务，瓶颈在于内存延迟和随机访问带宽。
*   **研究动机**：现有的同构存内计算（PIM）架构（如纯 PUM 或纯 PNM）难以同时高效处理这两种模式。本文旨在通过异构 PIM 芯片组（Chiplet）架构，实现硬件专业化与算法结构的精准匹配。

#### 2. 方法论
*   **核心思想**：设计一种 2.5D 封装的异构 PIM 加速器，将计算任务解耦到最适合的存储介质和处理单元上。
*   **关键技术细节**：
    *   **异构存储层次**：
        *   **热数据层（HBM3）**：用于 S2G 比对，提供高带宽和低延迟。
        *   **温数据层（PCM）**：用于 APSP，利用相变存储器的非易失性和高密度进行位串行计算。
        *   **冷数据层（FeNAND）**：用于大规模图数据的持久化存储。
    *   **Matrix-tile（PUM 瓦片）**：基于 PCM，执行位串行（Bit-serial）运算。采用**递归分区策略**将大图拆分为 1024 节点的子块，并设计了专门的排列单元（Permutation Unit）和比较器树以加速 Min-Plus 矩阵乘法。
    *   **Traversal-tile（PNM 瓦片）**：集成在 HBM3 逻辑层。采用**窗口位并行（Bit-parallel）算法**，通过三级存储层次（寄存器、模式缓存、共享 SRAM）处理复杂的拓扑依赖。

#### 6. 主要结论与发现

#### 7. 优点
*   **架构创新**：首次提出针对图 DP 两种冲突模式的异构 PIM 芯片组方案，打破了“一刀切”的硬件设计局限。
*   **精确计算**：与许多采用剪枝或近似算法的加速器不同，GEN-Graph 支持**精确的**动态规划更新。
*   **软硬协同**：递归分区算法与硬件 Tile 大小的深度耦合，使得大规模图计算能够高效映射到受限的 PIM 资源上。
*   **高能效**：利用 2.5D 封装和存内计算特性，大幅减少了数据在存储层次间的无效移动。

#### 8. 不足与局限
*   **仿真依赖**：目前研究基于周期精确模拟和 RTL 综合，缺乏实际流片（Tape-out）的物理测试数据，实际制造中的良率和 PCM 写入寿命（Endurance）可能面临挑战。
*   **应用范围**：虽然针对“通用”图 DP 模式，但主要针对的是能转化为矩阵或遍历模式的算法，对于更复杂的非线性 DP 逻辑支持尚不明确。
*   **硬件复杂性**：2.5D 封装集成 HBM、PCM 和逻辑芯片，制造工艺复杂且成本较高。
