Abstract 

On repetitive text collections of size n, the Burrows-Wheeler Transform ( BWT ) tends to have relatively fewer runs r in its run-length encoded BWT ( RLBWT ). This motivates many RLBWT -related algorithms and data structures that can be designed in compressed O(r)-space. These approaches often use the RLBWT -derived permutations LF , FL , ϕ , and ϕ−1, which can be represented using a move structure to obtain optimal O(1) -time for each permutation step in O(r)-space. They are then used to construct compressed space text indexes supporting efficient pattern matching queries. However, move structure construction in O(r)-space requires an O(r log r)-time balancing stage. The longest common prefix array ( LCP ) of a text collection is used to support pattern matching queries and data structure construction. Recently, it was shown how to compute the LCP array in 

O(n + r log r)-time and O(r) additional space from an RLBWT . However, the bottleneck remains the O(r log r)-time move structure balancing stage. In this paper, we describe an optimal O(r)-time and space algorithm to balance a move structure. This result is then applied to LCP construction from an RLBWT to obtain an optimal O(n)-time algorithm in O(r)-space in addition to the output, which implies an optimal-time algorithm for LCP array enumeration in compressed O(r)-space.

# 1 Introduction 

The Burrows-Wheeler Transform (BWT ) forms the basis of many efficient algorithms and applications in text compression, indexing, and pattern matching. When computed over a repetitive text collection, its run-length encoded BWT (RLBWT) contains relatively fewer runs r. Hence, many techniques have been developed in compressed O(r)-space [8, 13] which often grows sublinearily with respect to n. This is true for collections of DNA sequencing data, and thus the RLBWT is often used for applications in genomics. Indeed, the length of the text can reach the order of trillions on pangenome sized collections [11, 15, 19, 20] and thus RLBWT -based data structures are commonly used. Indexes built on repetitive texts rely on the RLBWT -based permutations LF , FL , ϕ and 

ϕ−1 to perform efficient pattern matching queries. Nishimoto and Tabei’s [13] move structure 

is able to represent these queries in O(r)-space while obtaining optimal O(1) query time for a single permutation step. Recently, Sanaullah et al. [15] showed how to compute the longest common prefix array (LCP ) from an RLBWT in O(n + r log r)-time and O(r)-space using move structures for these permutations. Their technique, which has many applications to matching algorithms and data structure construction, depends on the optimal-time guarantees of a single move query to achieve their time complexity. However, move structures require a balancing step to obtain constant-time queries. The best algorithm by Bertram, Fischer, and Nalbach [4] balances a move structure in O(r log r)-time and O(r)-space. Indeed, the O(r log r)-time factor in Sanaullah et al.’s [15] method is the result of balancing. Therefore, we present the first optimal O(r)-time and space move structure balancing algorithm. By analyzing Sanaullah et al.’s [15] approach with our improved construction, we also give the first optimal O(n)-time algorithm to compute the LCP array from the RLBWT in O(r)-space in addition to the output. These advancements improve the runtime of any application involving move structure balancing and LCP enumeration from an RLBWT .

# 2.3 Move Structure 

For a permutation π over {0, . . . , n − 1}, let P be the set of r distinct positions such that either i = 0 or π(i − 1) ̸ = π(i) − 1. For simplicity, let P [r] = n so that the domain of the permutation is captured by P . This defines a disjoint set of intervals, where, for any 

j ∈ P , the jth input interval with length ℓ = P [j + 1] − P [j] is defined as [P [j], P [j + 1]) .By definition of P , input intervals permute contiguously such that if position i belongs to the jth interval then π(i) can be computed as π(i) = π(P [j]) + ( i − P [j]) . However, finding which interval a given i belongs to would require a predecessor query on P and hence ω(1) 

query time [2]. If we assume that we know i and the rank of its predecessor in P , i.e its input interval, then Nishimoto and Tabei’s move structure [13] computes π(i) and the rank of its predecessor in P in O(r)-space and O(1) -time. Let Pπ = {π(i) | ∀ i ∈ P [0 ..r − 1] }. Then, knowing i

belongs to interval j, we can easily compute π(i) = Pπ [j] + ( i − P [j]) . However, to return the interval of π(i) without a rank query, they store Prank (π) = {P.rank (π(i)) | ∀ i ∈ P [0 ..r − 1] };intuitively, the input interval containing Pπ [j]. Since π(i) ≥ Pπ [j], we perform a linear scan of P starting from position Prank (π)[j] until we find the largest rank j′ such that P [j′] ≤ π(i)

to return the input interval containing π(i). This computation is referred to as a move query .Nishimoto and Tabei [13] explained how to balance the input intervals by splitting some of them such that the worst case scan was bounded, obtaining O(1) -time and O(r)-space to compute a move query. Bertram, Fischer and Nalbach [4] described an O(r log r)-time and 

O(r)-space balancing algorithm. Recently, Brown and Langmead [6] showed an alternative splitting regime called length capping which constructs in O(r)-time and O(r)-space a move structure achieving average O(1) -time queries. While useful for applications where queries can be amortized over n consecutive steps, such as when their permutations are formed from a single cycle, regular move structure balancing and the O(r log r)-time construction factor Brown, Sanaullah, Zhang, and Langmead 3

is required to obtain an optimal guarantee for less ideal access patterns. Move structures were created for use with RLBWT based permutations LF and ϕ as well as their respective inverses FL and ϕ−1. As such, they can be represented in space proportional to the number of BWT runs [13]. A particularly useful relationship exists between 

ϕ intervals and irreducible PLCP values [8]. The jth input interval of ϕ, [P [j], P [j + 1]) ,is such that PLCP [P [j]] is irreducible and corresponds to PLCP +[j]. Otherwise, for any 

i ∈ (P [j], P [j + 1]) , PLCP [i] is reducible. Thus, given PLCP + alongside a ϕ/ϕ−1 move structure, we can output PLCP [i] with respect to our current position i in the permutation using Corollary 1 in O(1) -time [16, 18]. 

# 2.4 LCP Array Computation 2.4.1 LCP Array from SA 

The first algorithm computing the LCP array from BWT or SA in optimal O(n)-time was an O(n)-space approach by Kasai et al. [10] using SA [0 ..n − 1] , ISA [0 ..n − 1] , and T [0 ..n − 1] .From i = 0 → n − 1 it compares suffix T [i..n − 1] with its lexicographically preceding suffix (the suffix starting at j = SA [( ISA [i] − 1) mod n]). The suffixes T [i..n − 1] and T [j..n − 1] 

are compared symbol by symbol until a mismatch occurs at T [i + ℓ + 1] ̸ = T [j + ℓ + 1] 

at which point we set LCP [ ISA [i]] = ℓ. The key insight is that if ℓ > 1 symbols match at iteration i, then at least ℓ − 1 symbols match in iteration i + 1 .Consider that, if ℓ > 1, then for j = SA [( ISA [i] − 1) mod n], T [i..n − 1] = cwu and 

T [j..n − 1] = cwv have a shared prefix cw of length ℓ where c is a single symbol and 

u̸ = v. Since T [j..n − 1] is the lexicographic predecessor of T [i..n − 1] , it follows that 

T [j + 1 ..n − 1] = wv is lexicographically smaller than T [i + 1 ..n − 1] = wu . By property of the suffix array, the lexicographically smallest suffix with the largest shared prefix for any suffix is its predecessor. Therefore, T [i + 1 ..n − 1] shares a matching prefix of length at least 

ℓ − 1, the string w, with its lexicographic predecessor since there exists a lexicographically smaller suffix matching this length. Thus, we can safely skip ℓ − 1 comparisons in iteration 

i + 1 when LCP [ ISA [i]] = ℓ. A consequence of this insight is that comparisons of equal symbols occur at most n times (since each one increments ℓ and i + ℓ is non-decreasing from 

0 → n − 1) and character comparisons of non-equal symbols occur at most n times (since each one increments i). Therefore at most 2n symbol comparisons occur, implying an overall 

O(n)-time algorithm. Notice that the access pattern to LCP is not sequential, and further that j = SA [( ISA [i]−1) mod n] can be replaced with j = ϕ(i). The ϕ algorithm by Kärkkäinen, Manzini, and Puglisi [9] is a variant of the algorithm of Kasai et al. [10] which constructs the PLCP array first using ϕ. This allows them to omit ISA by taking as input T [0 ..n − 1] and SA [0 ..n − 1] 

and constructing ϕ[0 ..n − 1] . Using j = ϕ(i) they make the same comparisons as prior, but now set PLCP [i] = ℓ sequentially as i is incremented. The final LCP array is then computed using the fact that LCP[ i] = PLCP[SA[ i]] .

# 2.4.2 LCP Array from BWT 

The above algorithms all rely on random access to the text T ; when stored in memory to achieve constant time, this requires n log σ bits. Beller et al. [3] introduced the first method to compute LCP from BWT or SA which attempts to use smaller working space, achieving 

O(n) additional bits and O(n log σ)-time. They avoid random access to T by working over a 

BWT and its wavelet tree. Prezza and Rosone [14] achieved o(n) bits of additional space by combining the approach of Beller et al. [3] with Belazzougui et al.’s [1] suffix tree interval 4 Optimal-Time Balancing and LCP from RLBWT 

enumeration algorithm. Although this does not require random access to T , it does require wavelet tree queries and the extra O(log σ)-time factor. Further, although compact, their space bound is not in terms of r which may be desired for large repetitive texts. Recently, Sanaullah et al. [15] showed how to compute the LCP array from the RLBWT 

in O(n + r log r)-time and O(r)-space. First, they utilize that only irreducible PLCP values are needed; when sampled alongside a ϕ/ϕ−1 move structure, the remaining PLCP values can be recovered. Second, given the recent development of move structures, they show how to simulate random access to T as required in the ϕ algorithm by using a move structure for 

FL . However, since a single step of the algorithm requires comparing the symbols T [i + ℓ]

and T [j + ℓ], where j = ϕ(i), it is not obvious how to find the correct position k = ISA [j + ℓ]

to perform FL queries without full access to ISA . Their key insight is that with a proper sampling of ISA values in O(r)-space, they can apply the ϕ algorithm using a move structure for FL in O(n)-time and O(r)-space space to compute the irreducible PLCP values. However, the overall time complexity of this approach inherits the O(r log r)-time con-struction of move structure balancing, resulting in O(n+r log r)-time and O(r)-space working space to compute the LCP array from an RLBWT . Although length capping [6] has shown that certain RLBWT operations involving O(n) consecutive move queries, such as suffix array enumeration and BWT inversion, can be done in optimal-time without balancing, the 

ϕ algorithm is not one of them. Therefore, to achieve optimal time complexity using these approaches, we need a move structure balancing algorithm taking linear-time.

# 3.2.2 Algorithm Description 

We proceed similarly to the balance-on-the-fly approach of Bertram, Fischer, and Nalbach [4], iterating left to right over the intervals. Heavy intervals are balanced during this scan, or when they are created by an insertion into an already-balanced portion of the intervals. However, since we balance left to right in both P and Q simultaneously, we maintain the 

balanced-up-to parameter t ∈ [0 , n ] initialized to 0 and incremented during balancing until we finish. This maintains the invariant that any intervals in P or Q starting before t must not be heavy up to either the end of their ranges or the position t.Brown, Sanaullah, Zhang, and Langmead 7

▶ Definition 16. Assume the balanced-up-to parameter is t. Then for arbitrary p ∈ P , if 

p.idx < t , then | (p.idx, min( t, p.next.idx )) ∩ Q| ≤ 2α. Similarly, for arbitrary q ∈ Q , if

We maintain pointers to pt ∈ P and qt ∈ Q which are the respective input and output intervals containing the current t. At each iteration, we select either pt or qt to balance, choosing the one with the minimum starting position. If tied, we choose the longer interval to balance, or proceed if they have the same length since by definition they must both be balanced. Then, we begin a balancing step by making the corresponding insertion using τ or

The intuition behind balancing in both directions is that we can obtain O(1) -time maintenance of predecessor pointers if the corresponding insertion using τ or τ −1 is at a position less than t. Otherwise, when inserting at a position greater than or equal to t, we maintain all pointers except predecessors. However, our left to right scan of the intervals can amend these as we reach them, since updating pt and qt requires following next pointers from the current nodes. During this update, we can recover predecessor information similar to the initialization of Lemma 15. # 5 Conclusion 

We have described the first optimal O(r)-time and O(r)-space move structure balancing algorithm. This improves on the previous best approach by Bertram, Fischer, and Nal-bach [4] using O(r log r)-time and O(r)-space complexity. As a result of combining the improved balancing complexity with the approach of Sanaullah et al. [15], we obtain the first optimal O(n)-time LCP computation algorithm from an RLBWT in O(r) additional space. Additionally, from an RLBWT , we obtain in O(r)-space: Optimal O(r)-time construction of balanced move structures for LF and FL ,

O(n)-time construction of balanced move structures for ϕ and ϕ−1,and an optimal O(n)-time algorithm for enumerating the LCP array. Since the balancing algorithm described here balances with respect to both π and its inverse π−1, a natural followup is to collapse both into a single move structure. That is, we store the interval starts in P ∪ Q. We call this an invertible move structure . Supporting move queries in both directions currently requires two move structures with O(r)-space additional satellite data for coordinate conversions. Possible benefits of the invertible move structure include common coordinates for π and π−1 move queries without satellite data, computing both π and its inverse in constant time, and removing the need for the “offset field” [5] in practical implementations. We leave detailed investigation as future work. A possible application of LCP computation from the RLBWT is Shivakumar and Lang-mead’s [17] mumemto tool which computes maximal match queries with respect to a collection of sequences. Given an RLBWT , its remaining bottleneck was LCP enumeration [6], where we stream the output rather than save the entire LCP array. Our improvements provide an 

O(n)-time and O(r)-space algorithm for mumemto which may be of practical interest. Finally, the current best known result for construction of the RLBWT from a given text in O(r)-space is Nishimoto, Kanda, and Tabei’s [12] O(n + r log r)-time algorithm. Therefore, the construction of move structures computing LF , FL , ϕ, ϕ −1 in O(r)-space from the text remains O(n + r log r)-time. This work removes the move structure balancing bottleneck from this time bound. Now, the only remaining obstacle to lowering these time complexities is improving upon the result for RLBWT construction in O(r)-space, or computing the
